What This Is

The Root Specification defines what ProtoSynthesis is at its highest level. It establishes the identity, scope, and boundaries of the entire system.

This is the entry-level definition of the architecture — everything else derives meaning and legitimacy from this layer.

System Identity

ProtoSynthesis is a layered systems architecture for structuring, modelling, and governing complex human and computational systems.

It is not a product, organisation, ideology, or application. It is an architectural system for organising systems.

What It Defines

  • What may exist within the ProtoSynthesis system
  • What is considered valid or invalid within the architecture
  • How systems relate at the highest structural level
  • How all subsystems derive legitimacy

System Scope

ProtoSynthesis applies to structured domains such as:

  • Knowledge systems
  • Learning and education systems
  • Capability and skill development
  • Organisational structures
  • Economic and value systems
  • Creative and media systems

It defines structure — not content.

Relationship to G01DN

G01DN defines how structural authority works inside the system. The Root Specification defines what the system is.

Together they form:

  • Root Specification → Identity
  • G01DN → Structure
  • Layers → Behaviour and rules

Key Principle

No implementation, file, or subsystem is the architecture itself. All systems are expressions of this root definition.
